# Shiora Health AI -- API Reference

## Base URL

```
Production: https://api.shiora.health
Development: http://localhost:3001/api
```

## Authentication

All API requests require a valid wallet signature in the `Authorization` header:

```
Authorization: Bearer <wallet-signed-jwt>
```

The JWT is generated by signing a challenge nonce with the user's Aethelred wallet private key. The server verifies the signature against the registered wallet address.

### Authentication Flow

1. `GET /api/auth/challenge` -- Receive a nonce
2. Sign the nonce with your wallet
3. `POST /api/auth/verify` -- Submit signature, receive JWT
4. Include JWT in all subsequent requests

## Error Codes

| Code | Description                      |
|------|----------------------------------|
| 200  | Success                          |
| 201  | Created                          |
| 400  | Bad Request -- Invalid parameters|
| 401  | Unauthorized -- Invalid/missing JWT |
| 403  | Forbidden -- Insufficient permissions |
| 404  | Not Found                        |
| 409  | Conflict -- Duplicate resource   |
| 422  | Unprocessable Entity -- Validation error |
| 429  | Too Many Requests -- Rate limited |
| 500  | Internal Server Error            |

### Error Response Format

```json
{
  "error": {
    "code": "RECORD_NOT_FOUND",
    "message": "Health record with the specified ID does not exist",
    "details": {}
  }
}
```

## Rate Limiting

| Endpoint Category | Rate Limit          |
|-------------------|---------------------|
| Read operations   | 100 requests/minute |
| Write operations  | 20 requests/minute  |
| TEE attestation   | 50 requests/minute  |
| Auth endpoints    | 10 requests/minute  |

Rate limit headers are included in every response:

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1704067200
```

## Health Records

### List Records

```
GET /api/records
```

**Query Parameters:**

| Parameter | Type   | Default | Description                           |
|-----------|--------|---------|---------------------------------------|
| type      | string | "all"   | Filter by record type (lab_result, imaging, prescription, vitals, notes) |
| search    | string | ""      | Search in label, provider, tags       |
| status    | string | "all"   | Filter by status (Verified, Pinning, Pinned, Processing) |
| sort      | string | "date"  | Sort field (date, label, size)        |
| order     | string | "desc"  | Sort order (asc, desc)                |
| page      | number | 1       | Page number                           |
| limit     | number | 20      | Records per page (max 100)            |

**Response:**

```json
{
  "records": [
    {
      "id": "rec-a1b2c3d4e5f6",
      "type": "lab_result",
      "label": "Complete Blood Count",
```

### Get Record

```
GET /api/records/:id
```

**Response:** Single record object (same schema as list item).

### Upload Record

```
POST /api/records
```

**Request Body:**

```json
{
  "type": "lab_result",
  "label": "Hormone Panel",
  "encryptedData": "<base64-encoded-encrypted-blob>",
  "provider": "Dr. Sarah Chen, OB-GYN",
  "tags": ["routine", "lab"]
}
```

**Response:** `201 Created` with the created record object.

### Delete Record

```
DELETE /api/records/:id
```

**Response:**

```json
{
  "success": true,
  "txHash": "0xabc123..."
}
```

## WebSocket Events

Connect to `wss://api.shiora.health/ws` for real-time updates.

### Event Types

| Event              | Description                        |
|--------------------|------------------------------------|
| `block.new`        | New block mined on Aethelred L1    |
| `record.uploaded`  | New health record uploaded         |
| `record.verified`  | Record TEE verification complete   |
| `access.granted`   | New provider access granted        |
| `access.revoked`   | Provider access revoked            |
| `access.used`      | Provider accessed health data      |
| `tee.attestation`  | New TEE attestation generated      |
| `anomaly.detected` | AI anomaly detection alert         |

### Event Format

```json
{
  "event": "block.new",
  "data": {
    "blockHeight": 2847392,
    "timestamp": 1704153600000,
    "tps": 2200
  }
}
```
